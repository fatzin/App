Sobre alguns Design Pattens utilizados no projeto.

Model-View-Controller (MVC): O Django segue um padrão de design chamado Model-View-Controller (MVC), que separa a lógica de negócios da interface do usuário e ajuda a manter o código organizado e fácil de manter.

Model-View-Template (MVT): O Django segue um padrão de design semelhante ao MVC, chamado Model-View-Template (MVT). Nesse padrão, a camada de controle é substituída pela camada de template, que é responsável por renderizar o HTML para o usuário final.

Object-Relational Mapping (ORM): O Django usa um ORM (Mapeamento Objeto-Relacional), que é uma técnica de programação que mapeia objetos em um programa para tabelas em um banco de dados relacional.

Front Controller: O Django usa um Front Controller, que é um padrão de design que centraliza o controle de solicitações em um único ponto de entrada, permitindo que todas as solicitações passem por uma camada de controle central antes de serem roteadas para o controlador apropriado.

URL Dispatch: O Django utiliza o padrão URL Dispatch para mapear URLs para views. Cada URL é mapeada para uma função de view que é responsável por retornar a resposta HTTP apropriada.

Middleware: O Django utiliza o padrão Middleware para adicionar funcionalidades extras ao processamento de solicitações e respostas HTTP. O middleware fica entre o servidor web e o aplicativo, podendo interceptar e processar as solicitações antes e/ou depois que elas chegam ao aplicativo.

Factory Method: O Django utiliza o padrão Factory Method para criar instâncias de classes em tempo de execução. Por exemplo, o Django usa a factory method para criar objetos de conexão com o banco de dados.

Dependency Injection: O Django usa o padrão Dependency Injection para injetar dependências em classes e funções, facilitando a criação de componentes com dependências claras e gerenciáveis.

Observer: O Django utiliza o padrão Observer para notificar os objetos de mudanças em outros objetos. Por exemplo, o Django usa o padrão Observer para notificar os modelos (models) quando as tabelas do banco de dados são alteradas.

Representational State Transfer (REST): O padrão REST é uma arquitetura de software que define um conjunto de princípios para projetar APIs web. Em um sistema RESTful, as solicitações HTTP são mapeadas para recursos (resources) que podem ser criados, lidos, atualizados e excluídos (CRUD). O REST é amplamente utilizado em APIs web modernas e é suportado nativamente pelo Django através do framework Django REST.

Sobre o projeto final

O projeto final foi usado utilizando um simples banco de dados para o processamento de informações na aba URL.

Para o segundo app do projeto chamado de temperatura foi utilizado solicitações HTTP para o consumo da API do OpenWeatherMap.

Para o segundo app do projeto chamado de Moedas que é um conversor de moedas foi utilizado solicitações HTTP para o consumo da API da ExchangeRate-API.
